Abstract

Fine-tuning large language models (LLMs) traditionally requires updating billions of parameters, demanding prohibitive GPU memory and compute. Parameter-Efficient Fine-Tuning (PEFT) methods such as LoRA and QLoRA address this by training only a tiny fraction of adapter weights. In this study we empirically compare three strategies on the IMDb binary sentiment classification task: zero-shot inference, LoRA fine-tuning, and QLoRA (4-bit NF4 quantised LoRA). LoRA achieved an F1 of 0.9337 vs. 0.9337 zero-shot, while QLoRA delivered 0.9458 F1 with substantially lower GPU memory consumption.

1. Introduction

The rapid growth of LLM parameter counts — from hundreds of millions to hundreds of billions — has made full fine-tuning prohibitively expensive for most practitioners. PEFT methods inject small trainable modules into frozen pre-trained models, reducing trainable parameters by orders of magnitude while preserving downstream task quality.

LoRA (Hu et al., 2022) decomposes each weight update into a product of two low-rank matrices (rank r), limiting trainable parameters to r×(din+dout) per targeted layer. QLoRA (Dettmers et al., 2023) extends LoRA by quantising the frozen base model to 4-bit NF4, enabling fine-tuning of 65B-parameter models on a single 48 GB GPU.

5. Discussion

The zero-shot F1 of 0.9337 reflects Qwen/Qwen2.5-1.5B-Instruct's pre-trained instruction-following capability without any task-specific signal. LoRA fine-tuning improved F1 to 0.9337 by training only 0.141% of total parameters. QLoRA achieved 0.9458 F1 while consuming 4646 MB of peak GPU memory — demonstrating that 4-bit quantisation introduces minimal accuracy degradation for classification tasks.

6. Conclusion

Both LoRA and QLoRA substantially outperform zero-shot inference on IMDb, validating PEFT as a practical, memory-efficient alternative to full fine-tuning on resource-constrained hardware. QLoRA in particular is recommended for practitioners with ≤16 GB VRAM, delivering near-LoRA accuracy at a fraction of the memory cost.

7. Future Work

  • Merge LoRA adapters into base weights for zero-overhead inference
  • Extend benchmarks: NLI, QA, abstractive summarisation
  • Apply GPTQ / AWQ post-training quantisation for inference acceleration
